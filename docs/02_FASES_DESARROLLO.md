# Fases de Desarrollo

## Descripcion

Este documento define las fases iniciales de desarrollo de Con Sabor Flamenco. El objetivo es avanzar de forma ordenada, evitando implementar funcionalidades complejas antes de que la base documental, visual y tecnica este clara.

## Objetivo del documento

Establecer una hoja de ruta inicial para el proyecto y documentar que cada fase debe tener un alcance concreto, revisable y ampliable.

## Fases iniciales

### Fase 0 - Documentacion inicial

Crear la documentacion base del proyecto, definir vision, areas principales, reglas comerciales, fases y decisiones tecnicas iniciales.

### Fase 1 - Estructura visual y menu principal

Definir la estructura visual inicial, navegacion publica y menu principal.

Alcance documentado para esta fase:

- Crear o ajustar el header publico.
- Definir el menu principal de la web publica.
- Crear una primera home tipo revista/comunidad.
- Preparar secciones publicas de ejemplo.
- Crear o ajustar el footer publico.
- Definir estilos globales iniciales sin logica compleja.

### Fase 2 - Diseno de home tipo revista y monetizacion publicitaria

Crear una home moderna con enfoque editorial, contenidos destacados y acceso a las principales secciones.

Alcance publicitario incorporado:

- Sidebar derecho reutilizable para las paginas publicas.
- Selector inicial de provincia para visitantes.
- Preferencia de provincia persistente en el dispositivo.
- Integracion futura con la provincia del perfil registrado.
- Banners segmentados por provincia y categoria del navbar.
- Campanas generales o nacionales de respaldo.
- Preparacion documental del modelo de campanas, impresiones y clics.

Alcance visual incorporado:

- Rankings editoriales reutilizables para las categorias activas.
- Moda flamenca con tarjetas para Ropa, Calzado, Complementos y Moda infantil.
- Paginas publicas independientes para Servicios y Flamenco.
- Concursos conservado en codigo y temporalmente oculto.

### Fase 3 - Modelo inicial de base de datos

Definir las primeras tablas, relaciones y criterios de datos para miembros, contenidos, servicios, leads e inbox.

### Fase 4 - Panel de administracion

Crear la base del area de administracion como centro de control de la plataforma.

### Fase 5 - Gestion de miembros

Implementar gestion administrativa de miembros y tipos de miembros.

### Fase 6 - Perfiles publicos de miembros

Crear perfiles visibles para artistas, academias, penas, tablaos, festivales, profesionales y entidades colaboradoras.

### Fase 7 - Area de appointment setters

Crear el panel privado de setters con acceso a su informacion comercial.

### Fase 8 - Codigos promocionales

Implementar el sistema de codigos promocionales asociados a appointment setters.

### Fase 9 - Leads, ventas y comisiones

Crear el flujo comercial desde lead hasta venta y calculo de comisiones.

### Fase 10 - Solicitudes de cobro y liquidaciones

Gestionar solicitudes de cobro, comisiones pagadas, liquidaciones y justificantes internos.

### Fase 11 - Inbox y agente IA

Crear la inbox de administracion y preparar la integracion del agente IA y formularios inteligentes.

### Fase 12 - Servicios digitales y planes

Definir y gestionar servicios digitales, planes, contrataciones y estados de servicio.

### Fase 13 - API REST y preparacion para app movil

Preparar endpoints, contratos de datos e integraciones necesarias para futuras apps o sistemas externos.

## Reglas y decisiones

- Cada fase debe tener un alcance claro.
- No se debe implementar funcionalidad avanzada antes de documentar su base.
- La documentacion debe actualizarse al cerrar cada fase.
- Las fases pueden ajustarse si el proyecto lo necesita, dejando constancia en el historial.

## Pendiente de definir

- Criterios de aceptacion por fase.
- Prioridades comerciales exactas.
- Calendario de desarrollo.
- Dependencias tecnicas entre fases.
